# pet-medicine-app

ペットの投薬管理アプリ。ペットごとに薬の種類・用量・投与スケジュールを記録・管理する。

## Git 運用ルール

**コードを変更するたびに GitHub へプッシュする。**

```
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin <ブランチ名>
```

- コミットメッセージは日本語でも英語でも可。変更の「なぜ」を一言で表す。
- `main` ブランチへの直接プッシュは避け、機能ブランチ → PR → マージのフローを使う。
- 破壊的な操作（`--force` push、`reset --hard` 等）は実行前に必ず確認を取る。

## 禁止事項
- `rm -rf` コマンドは絶対に実行しない
- `.env` ファイルを読んだり変更したりしない
- `package.json` の依存パッケージを無断で変更しない
- データベースへの削除操作（DELETE文）を実行しない

## 開発方針

- シンプルさを優先し、不要な抽象化・将来対応の機能は追加しない。
- コメントは「なぜ」が自明でない箇所にのみ書く。
- セキュリティ: ユーザー入力は必ずバリデーションする。SQL インジェクション・XSS に注意。

## 実施内容の確認ルール
- 実行前の確認が入る際、実行内容・実行時の影響やリスクを提示した上で確認する。
- 作業の中で、セキュリティーに影響やリスクがあるコードやコマンドを実行する際は、必ずその内容を説明し、実行前に確認する。
